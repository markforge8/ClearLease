ClearLease · Explain v2
变更与使用规则（冻结条款）

本规则用于约束 Explain v2 的设计、演进与调用方式，
目的是保证判断一致性、可解释性与长期稳定性。

一、Explain v2 的定位（不可更改）

Explain v2 是判断解释模块，不是决策模块

Explain v2 不提供法律、医疗、财务建议

Explain v2 的职责仅限于：

解释已识别的结构性风险机制

描述当前所处状态

指出是否存在行动窗口（escape window）

❌ Explain v2 不负责“该不该做”
✅ Explain v2 只负责“你现在处在什么结构里”

二、机制判断规则（硬约束）

已存在的机制（如 Temporal Lock-in）不得被重新定义或修改语义

新机制只能新增，不得通过修改旧机制“曲线扩展”

单个样本不得触发机制定义调整

所有机制调整必须以「多样本重复失配」为前提

三、输出结构规则（Schema 稳定性）

Explain v2 的 Output Schema 字段名 视为对外契约

不得因 UX、文案或单一场景需求修改字段结构

escape_window 字段 必须始终存在：

即使窗口已关闭，也必须明确标注状态

四、表达与语气约束（非常重要）

Explain v2 的表达必须：

中性

克制

不带价值判断

禁止使用：

情绪化词汇

指责性表述

“陷阱 / 坑 / 割 / 骗”等判断性语言

解释应描述结构，不评价动机

五、版本与变更规则

Explain v2 的任何变更必须：

标注版本号（v2.x）

明确变更原因

以下情况才允许升级版本：

新机制加入

Schema 扩展

明确的系统性误判被证实

文案优化 ≠ v2 版本变更
文案优化属于 Gateway / UX 层

五.1、语言版本约束（v2.1+）

Explain v2.1 的默认语言已经收敛并冻结

后续所有 Gateway 映射仅使用 `explain_v2_1_default_language.md`

不再生成或建议新的语言版本

任何语言相关的需求必须基于冻结版本，不得新增变体

语言修改需要版本升级（v2.1.x），不属于日常优化范围

六、禁止事项（红线）

Explain v2 不得：

因品牌特殊化输出

因用户身份差异改变判断

因商业目标弱化风险表述

Explain v2 必须对所有输入保持结构一致性

七、最终原则（最高优先级）

当“更好看”和“更一致”冲突时，
永远选择一致性。
